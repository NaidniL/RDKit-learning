## 1. 项目状态总览

### 1.1 研究目标与当前结论

本项目以论文 *Toward Explainable Carcinogenicity Prediction: An Integrated Cheminformatics Approach and Consensus Framework for Possibly Carcinogenic Chemicals*（JCIM, 2025）为方法学起点，研究由分子结构预测二元致癌性（`1=carcinogen`、`0=noncarcinogen`）。它复现多表征、共识、适用域和解释性分析等核心思想，并在数据治理、外部隔离和可审计评估方面重新设计和扩展；因此不是对原论文数据、模型选择流程和外部评估的严格复现。项目重点是建立一条可审计、可复算、能限制外部集泄漏的 RDKit 毒理机器学习工作流。

当前有两条严格区分的成果线：

| 版本 | 状态 | 可以作出的结论 | 不可以作出的结论 |
|---|---|---|---|
| **v1：单模型可信基线** | 已冻结、独立审核、完成一次 primary external 评估 | `LightGBM + RDKit/physicochemical descriptors` 是在预注册候选和固定 validation 中选出的最终模型；其外部泛化指标已锁定 | 不能用外部结果回调参数、特征、阈值或再次挑选模型 |
| **v2：selective consensus** | 已冻结 policy、三成员 final artifact、NTP independent external 和最终 outcome | 该系统在 covered subset 有较高 MCC/accuracy/specificity，且 inconclusive 富集成员错误；最终状态为 `PARTIAL_SUCCESS` | 不能称为“验证成功”“优于 v1”或“可用于致癌物筛查”；不得再次读取 NTP candidate、重跑或以结果改动 policy/模型 |

v1 的最终外部结论为：在 455 个与 development 集不存在 exact InChIKey 或 connectivity-key 重叠的 CCRIS 化合物上，AUROC 为 **0.719437**、AUPRC 为 **0.772798**、MCC 为 **0.287757**、accuracy 为 **0.643956**。该隔离不等于所有 tautomer、scaffold 或高相似近邻关系均为零。这是一条可信基线，而非“已证明的最佳致癌性模型”。

### 1.3 工作流

```text
CPDB / IRIS / CCRIS 原始记录与结构
        ↓（来源专属清洗；保留原始证据）
RDKit 结构标准化、parent 提取、canonical SMILES / InChIKey
        ↓
严格标签映射、角色内聚合、冲突/不确定/结构异常审计
        ↓
正式 dataset release + manifest + SHA-256 + 泄漏审计
        ↓
development：train 内 5-fold 调参 → 固定 validation 一次性候选选择
        ↓
冻结 final config → train+validation refit → 独立审核
        ↓
一次性 primary external 评估（CCRIS；禁止回流）
        ↓
v1 的事后解释、误差与适用域描述
        ↓
v2 development-only 共识/不确定性研究
        ↓（冻结 policy、final-refit artifact；禁止 v1 CCRIS external）
NTP candidate 的 exact/connectivity/tautomer 零重叠审计
        ↓（独立授权、预创建 one-shot 输出锁）
一次性 NTP external evaluation → aggregate-only final outcome release
```

| `scripts/audit_v2_*`、`finalize_v2_evaluation.py`、`audit_v2_final_outcome.py` | v2 policy/artifact/external/outcome 审核与冻结 | final outcome 仅基于 aggregate manifest/report；不读取 external 行 |

训练接口有两项特别重要的防护：

1. 训练代码只能从 `releases/dataset_assembly/current_release.json` 定位正式 release，并验证 manifest；禁止手写 `data/splits/...` 路径。`current_release.json` 只是可变的解析入口；训练 manifest 还必须记录解析得到的不可变 release ID、manifest SHA-256 与 artifact hashes。冻结模型的复算应依赖后者，而不能依赖未来可能改变的 pointer。
2. 禁止将来源、角色、证据类型、来源 ID、标签、fold、审查/泄漏状态等字段作为特征。特征只来自 `canonical_smiles` 或 `parent_smiles`。

## 2. 数据集：来源、处理、冲突与划分

### 2.1 最终正式数据的来源与角色

| 来源 | 在本项目中的角色 | 标签资格（仅限明确二分类证据） | 最终用途 |
|---|---|---|---|
| CPDB（Carcinogenic Potency Database） | development | `+`、充分试验的 `c` 为 1；充分试验的 `-` 为 0 | 与 IRIS 合并为开发池 |
| IRIS（EPA Integrated Risk Information System） | development | 在本项目预定义 ontology 中，`A/Human carcinogen` 映射为 1，`E/Evidence of non-carcinogenicity for humans` 映射为 0 | 与 CPDB 合并为开发池 |
| CCRIS（Chemical Carcinogenesis Research Information System） | external | carcinogenicity 结果必须严格等于 `POSITIVE` 或 `NEGATIVE` | 一次性 primary external 测试 |
| ISSCAN | 非正式 release 成员 | 项目保留导入脚本和论文对照材料，但未纳入本次 v1 formal release | 可作为未来论文复现或对照数据集；是否能作为新的独立 external，取决于其是否未参与新项目的规则制定、模型选择和人工判断，并需另行进行重叠与访问审计 |

所有来源记录被保留在 `records_all.csv`，而不是只留下最终入模样本。正式 release 共保存 **38,686** 条来源记录、**8,267** 个结构实体和 **9,387** 条 `compound × dataset_role` 解析记录。这使每一个训练或排除决定都能回溯到来源证据。上述二元标签是“按本项目预定义证据规则解析出的标签”，并不等同于在任意剂量、物种、暴露途径或实验条件下对真实因果致癌性的绝对断言；CPDB、CCRIS 的 0/1 标签亦应作相同理解。

### 2.2 处理过程与结构标准化

处理顺序如下：

1. **来源专属字段与标签清洗**：保存 `label_raw`、来源记录 ID、原始结构和证据文本；未知或带限定语的标签不能默认转成阴性。
2. **RDKit 结构检查与标准化**：解析 SMILES，提取 parent，生成 canonical SMILES、标准 InChIKey、14 位 `connectivity_key`、non-isomeric parent、tautomer family 和 Murcko scaffold。
3. **结构资格判定**：盐/对离子、混合物、无有效小分子、无机含碳边界化合物、未解析来源或多 parent 表示等均记录具体原因。疑似无机含碳结构有人工审核表，不以名称猜测替代结构规则。
4. **角色内标签解析**：development 只用 CPDB/IRIS；external 只用 CCRIS。跨角色证据永不投票，因此同一结构在两个角色中即使标签不同，也不相互改写。
5. **结构关系与泄漏检查**：除精确 InChIKey 外，还检查 connectivity、互变异构体、Murcko scaffold 与 ECFP4 相似性；将关系与排除理由保存为独立 artifact。
6. **事务式发布**：先生成 dry-run audit，再仅在输入指纹、运行环境和所有 artifact 完全匹配时生成 formal release；current pointer 指向唯一训练输入。

清洗日志中的“严格开发集 864、冲突 613”等数字属于阶段 1 的中间冻结视图；最终建模必须以阶段 2 formal release 为准。后者按批准的冲突裁决规则形成 **921** 个 eligible development 样本、**455** 个 eligible external 样本，因而两组数字不应混用。

### 2.3 合并时的冲突与解决策略

| 问题 | 风险 | 解决办法 | 正式审计结果 |
|---|---|---|---|
| 多来源的标签冲突 | 同一结构同时有明确阳性、明确阴性；直接多数投票会掩盖证据质量 | 先按角色聚合；只计 `clear_positive_count` 与 `clear_negative_count`。若总明确记录数 ≤10 则排除；否则若两者差 ≤10 仍排除；仅在前两条不成立时按记录数较多一方赋标签 | 614 个冲突：451 个因总数不足排除、87 个因差距不足排除、51 个自动赋阳性、25 个自动赋阴性；所有结论在 `conflict_reviews.csv` 可追溯 |
| 弱证据/非二分类端点 | 把 `equivocal`、`inadequate`、可能/提示性证据误映射为 0 或 1，制造标签噪声 | 非明确致癌性端点标为 uncertain，不进入主 split；弱证据不能推翻明确证据 | 保留来源事实和 `discordant_nonclear_evidence` 标记，而不是删除证据 |
| 结构表示冲突 | 一个标准键对应多个 parent SMILES，随机选一个会改变特征 | 将多 parent 的已声明冲突标为 `structure_status=ineligible`，保留全部变体 JSON；未知新冲突直接失败 | 4 个 structure conflict |
| 盐、混合物、无机/边界碳化合物 | 特征代表的不是可定义的有机母体；隐性筛选不可审计 | parent/片段规则 + 显式排除理由；疑似无机含碳项目人工审核 | 早期清洗记录保留 44 项人工审核决定；正式 release 保留来源级与结构级排除表 |
| development 与 external 重叠 | 外部“泛化”会变成训练记忆或近重复检测 | external 遇到 development 的 exact InChIKey 或 connectivity key 即判 `ineligible_leakage`；tautomer overlap 不改变 primary 外部成员但单列报告并生成敏感性集 | 审计识别出 **1,120 条 external role 的 exact-overlap 状态**及 **141 条 external role 的 connectivity-overlap 状态**；这是 `compound × external role` 状态计数，不是 unique compounds 或 pair 数。primary external 的这两类交集均为 0 |

这里的 count 单位是已标准化的**来源记录**（`source_record`），不是独立研究、独立 assay、物种或数据库数；同一来源的高记录密度可能在第三步裁决中占更大权重。阈值 10 是在建模前写入冻结策略的资格门槛，用于只在明确来源记录数足够且方向差距足够大时自动赋标签，而非声称记录数等于独立证据强度。这是当前标签策略的限制。

上述冲突策略的关键取舍是**宁可减少可训练样本，也不把不确定标签、结构歧义或跨角色信息泄漏伪装成干净样本**。代价是数据量与化学空间覆盖度下降；收益是最终性能解释更可追溯，尤其适合外部评估和后续的不确定性输出。

### 2.4 训练、验证、外部验证与交叉验证划分

| 数据部分 | 样本数（阳性/阴性） | 产生方式 | 允许用途 |
|---|---:|---|---|
| train | 736（334 / 402） | eligible development 按标准 InChIKey 排序后，`train_test_split(test_size=0.2, stratify=y, random_state=42)` | train 内 5-fold 调参、拟合候选模型 |
| fixed validation | 185（84 / 101） | 同一次分层 80:20 的 20% holdout | 只承担一次冻结的候选选择决策；之后可为复算/审计重现预测，但不得据此追加候选、修改参数、阈值或选择规则 |
| final refit | 921（418 / 503） | train + validation | 仅用于已冻结 v1 配置的最终重训 |
| primary external（CCRIS） | 455（256 / 199） | 独立角色，已阻断 exact/connectivity overlap | 冻结后一次性评估；不得调参/调阈值/选模型 |
| external tautomer-clean sensitivity | 455（256 / 199） | 互变异构体敏感性视图 | 当前与 primary external 的成员和标签完全相同：1,182 条跨角色 tautomer-overlap 报告均不属于 primary external，故没有样本可移除；当前未纳入 v1 summary 的正式外部结论 |

另外提供三套彼此不能混用的 5-fold 文件：

- `train_tuning_cv_folds.csv`：只覆盖 736 个 train，用于超参数选择；
- `full_development_stratified_cv_folds.csv`：921 个 development，用于内部的论文式复现/稳健性描述；它覆盖已参与模型开发和选择的样本，不能替代独立 external；
- `full_development_scaffold_cv_folds.csv`：按 Murcko scaffold 或无环时 connectivity 的连通组分划组，用于内部、更严格的化学空间稳健性描述；它同样不是独立确认性证据。

fixed validation 是随机分层 holdout，且保留了 4 个 train/validation connectivity overlap 的报告事实；这不是 external 泄漏，但说明其指标主要衡量同一总体分布内的泛化，不能视为 scaffold-disjoint 或严格结构外推性能。外部隔离则采用 development 全集阻断，要求 primary external 与 development 阻断集合的 exact/connectivity 交集均为零。

## 3. v1 与 v2 的模型结果及最优模型选择

### 3.1 v1：预注册候选的固定 validation 结果

所有候选用相同的正式 release 和 train 内固定 5-fold 网格搜索；`external_access=denied`。主选择指标预注册为 validation AUROC，其后才是 AUPRC、MCC、sensitivity、specificity；AUROC 完全相同时才用特征简单性、预处理量、CV 方差打破平局。

| 排名 | 模型 + 特征 | train-CV OOF AUROC | validation AUROC | AUPRC | MCC | Sens. | Spec. | TN / FP / FN / TP |
|---:|---|---:|---:|---:|---:|---:|---:|---|
| 1 | LightGBM + descriptors | 0.8017 | **0.7659** | 0.7896 | 0.3806 | 0.6786 | 0.7030 | 71 / 30 / 27 / 57 |
| 2 | LightGBM + mixed | 0.8018 | 0.7653 | 0.7896 | 0.3588 | 0.6667 | 0.6931 | 70 / 31 / 28 / 56 |
| 3 | RandomForest + MACCS | 0.8057 | 0.7515 | 0.7636 | 0.3883 | 0.6548 | 0.7327 | 74 / 27 / 29 / 55 |
| 4 | RandomForest + descriptors | 0.8136 | 0.7514 | 0.7741 | **0.3968** | 0.6190 | 0.7723 | 78 / 23 / 32 / 52 |
| 5 | RandomForest + mixed | **0.8207** | 0.7512 | 0.7736 | 0.3856 | 0.6071 | 0.7723 | 78 / 23 / 33 / 51 |
| 6 | LightGBM + MACCS | 0.8015 | 0.7228 | 0.7439 | 0.3769 | 0.6429 | 0.7327 | 74 / 27 / 30 / 54 |
| 7 | RandomForest + ECFP4 | 0.8012 | 0.7210 | 0.7341 | 0.3744 | 0.5952 | 0.7723 | 78 / 23 / 34 / 50 |
| 8 | LightGBM + ECFP4 | 0.7844 | 0.6973 | 0.7014 | 0.2853 | 0.6429 | 0.6436 | 65 / 36 / 30 / 54 |

“mixed”是 ECFP4、MACCS 和 descriptors 的组合。这里的最高 train-CV OOF 并未决定最终模型，避免把 train 内表现当成独立选择证据。

### 3.2 v1 最优模型、冻结配置与外部结果

最终模型为 **LightGBM + RDKit/physicochemical descriptors**。其冻结参数为 `num_leaves=31`、`learning_rate=0.03`、`n_estimators=300`、`min_child_samples=10`、`reg_lambda=0`、`random_state=42`、`n_jobs=1`、`deterministic=true`，阈值固定为 `0.5`。输入 220 个 descriptor，经 final-refit median imputation 后删除 11 个常数列，保留 209 个特征。

| 指标 | fixed validation（185） | primary external（455） |
|---|---:|---:|
| AUROC | 0.765912 | 0.719437 |
| AUPRC | 0.789600 | 0.772798 |
| Accuracy | 未在最终汇总表单列 | 0.643956 |
| Balanced accuracy | 未在最终汇总表单列 | 0.644993 |
| MCC | 0.380600 | 0.287757 |
| Sensitivity | 0.678600 | 0.636719 |
| Specificity | 0.703000 | 0.653266 |
| TN / FP / FN / TP | 71 / 30 / 27 / 57 | 130 / 69 / 93 / 163 |

validation AUROC 比 external 高 **0.046475**。该下降与来源、标签本体、样本组成和化学空间差异可能造成的泛化损失相一致，但仅凭两个点估计不能判断该差异是否具有统计显著性，也不能将其完全归因于 applicability domain 或分布转移；候选选择带来的 validation 乐观偏差与随机抽样波动也是可能解释。事后分层提供支持性线索：external 的 near-domain（125 个）AUROC/MCC 为 0.799/0.363，out-of-domain（319 个）为 0.667/0.229。最小的 in-domain 分层仅 11 个样本，点估计为 1.000/1.000，缺乏稳定性，不应用于估计真实部署性能。

### 3.3 v2：三模型共识的开发期结果

v2 固定了三个不同表征模型，而没有用 v1 外部集来挑选它们：

| v2 成员 | 原始注册特征数 | 预处理后实际特征数 | fixed-validation AUROC | AUPRC | MCC |
|---|---:|---:|---:|---:|---:|
| LightGBM + descriptors | 220 | 209（训练集 median imputation 后移除 11 个常数列） | 0.765912 | 0.789603 | 0.380601 |
| RandomForest + MACCS | 167 | 167（无 descriptor 预处理/常数列移除） | 0.751532 | 0.763637 | 0.388306 |
| RandomForest + ECFP4 | 2048 | 2048（无 descriptor 预处理/常数列移除） | 0.710632 | 0.726311 | 0.385403 |

规则为：三者概率都 `>=0.5` 才输出 carcinogen；都 `<0.5` 才输出 noncarcinogen；其余输出 **inconclusive**。在 185 个 validation 样本中，明确调用覆盖率为 **134/185 = 0.724324**，51 个（27.57%）被拒答；已覆盖子集 accuracy 为 **0.761194**、MCC 为 **0.525774**（TN/FP/FN/TP=58/11/21/44）。

unanimity 规则在 72.43% 的已覆盖样本上取得 MCC 0.525774，高于各单模型在**完整** validation 上的 MCC（约 0.38）。两种指标的样本集合不同：被拒答的 51 个样本可能本来就更困难，因此这个比较不能单独证明 consensus 优于成员模型。要评估增益，还须比较每个成员模型在同一 covered subset 上的表现、拒答子集的错误富集，以及不同 coverage 下的 selective risk/risk-coverage 曲线。两两模型仍存在各自独有错误（例如 descriptor 与 MACCS 的错误 Jaccard 为 0.506667）；这只说明当前 fixed validation 的错误并非完全重合，为进一步研究表征互补和 disagreement-based abstention 提供经验依据，尚不能证明化学机制互补、外部提升或“分歧天然等于不确定性”。将 inconclusive 作为显式输出在工程上可行，但其实际筛查价值仍须结合漏检成本、人工复核能力和独立 external 结果评估。

### 3.4 v2：独立 NTP external、最终分类与解释边界

v2 随后冻结了三成员 final-refit artifact，并使用 **NTP/NICEATM Cancer Bioassay Chemicals** 的新候选 external。候选标签在读取模型前固定：任一 `P` 且无 `NE` 记为 carcinogen；至少一个 `NE` 且所有观察值仅为 `NE`/`NT` 记为 noncarcinogen；冲突或不确定证据排除。候选相对 921 个 formal development 样本的 exact、connectivity 和 tautomer overlap 均为零；v1 CCRIS primary external 未被读取或复用。

评估由单独 authorization 绑定 policy、final artifact 和 candidate release manifest 的 hashes，并先创建 one-shot 输出锁。它只执行一次，未保存样本级 prediction CSV；随后 independent complete audit 验证了输出文件集合、授权、三种 overlap、immutable final artifact、AD 的 report-only 边界和无事后改模。

| 外部结果 | 数值 / 解释 |
|---|---|
| NTP external | 49 个样本：15 carcinogen、34 noncarcinogen |
| v2 calls | 6 carcinogen、26 noncarcinogen、17 inconclusive；coverage `0.653061` |
| class-conditional coverage | carcinogen `0.533333`；noncarcinogen `0.705882`，存在必须报告的类别不平衡 |
| covered subset | 32 个；MCC `0.647150`、accuracy `0.875000`、sensitivity `0.625000`、specificity `0.958333`；TN/FP/FN/TP=`23/1/3/5` |
| same-covered-subset 比较 | 三个 v2 成员与 consensus 在该 unanimity subset 的 hard call 相同；v2 MCC 与 v1 在同一 subset 的 MCC 相等（`0.647150`），只满足“≥”规则，不构成优于 v1 的证据 |
| 错误拦截 | 成员平均错误率：covered `0.125000`，inconclusive `0.392157`；17 个 inconclusive 中各成员聚合错误数为 descriptor 9、MACCS 5、ECFP4 6 |
| 预注册判定 | coverage、同 subset MCC、错误富集和 specificity 通过；sensitivity 未通过（`0.625000 < 0.650000`） |

因此最终 outcome JSON 机械锁定为 **`PARTIAL_SUCCESS`**，而不是完整成功：

```text
final tag: final-consensus-v2-external-partial-v1
overall_success: false
failure reason: covered sensitivity 0.625000 < preregistered minimum 0.650000
```

覆盖 sensitivity 的 Wilson 95% 区间为 5/8，即 `[0.305742, 0.863156]`；specificity 的区间为 23/24，即 `[0.797582, 0.992607]`。它们只说明 covered positives 很少、点估计不稳定，不能替代预注册的点估计门槛，也不能将 sensitivity criterion failed 改写为通过。

故最准确的结论是：**v2 selective consensus system：独立 external 验证部分成功。** 它支持“分歧可作为错误富集/人工复核信号”的研究假设，但不满足一个固定致癌性筛查策略的所有预注册条件。

## 4. 与原文献的继承、分歧与结果含义

### 4.1 原文献的可迁移思想与本项目的采用方式

| 原文献思想 | 本项目的采用 |
|---|---|
| 用多种分子表征和模型观察互补性 | v1 比较 descriptors、MACCS、ECFP4 与 mixed；v2 固定 descriptor/MACCS/ECFP4 三模型研究分歧 |
| LightGBM、RF、BiLSTM 作为代表性学习器 | v1 实现并评估 RF、LightGBM（也保留 Logistic/Dummy 与 BiLSTM 环境基础）；冻结选择依据是预注册 validation 规则 |
| 固定 80:20 分层划分、随机种子 42、5-fold CV | primary split 延续该对照协议；但将 train 内 CV、fixed validation、full-development CV 和 scaffold CV 分离职责 |
| 共识允许不一致结果为 inconclusive | v2 复用“全体同意才出结论”的思想，显式报告 coverage 与 covered-set performance |
| 用结构相似度/适用域约束解释预测 | 保存 ECFP4、最近邻、scaffold、tautomer 关系；v1 外部和 v2 validation 都有描述性 AD 分层 |
| SHAP 与结构化解释 | v1 对可解释的 descriptor LightGBM 做 SHAP、方向性及错误 descriptor profile；强调相关性不是机制因果 |

### 4.2 关键分歧、选择理由与已观察到的优劣

| 主题 | 原文献 | 本项目的选择及理由 | 结果中的优势与代价 |
|---|---|---|---|
| 数据组成 | 训练集为 CPDB、CCRIS、ISSCAN 的整理组合，1,599 个化合物；外部有 ISSCAN 40 个和 IRIS/IARC 人类致癌集 73 个（论文 §2.1–2.2，pp. 10196–10197） | development 固定为 CPDB+IRIS，CCRIS 独立为 external；ISSCAN 未进入 formal release。目的是将 external 的来源角色与标签证据隔离，避免同源记录和跨角色“确认” | external 样本数为 455，且数据边界可审计；但数据构成不同，不能把本项目指标与论文的 ISSCAN/IARC 数值作头对头优劣比较 |
| 标签冲突 | 文献描述缺失/重复检查和数据清理，但未提供本项目这种逐角色、计数阈值化的冲突裁决链 | 明确/弱证据分离；跨角色绝不投票；614 个冲突按预写三步规则处理 | 降低标签噪声和事后人为选择；代价是排除大量记录、模型可见化学空间变窄 |
| 外部重叠 | 文献的 IRIS/IARC 外部集中保留与训练集的冗余分子，只从训练集中移除冗余（论文 §2.2，p. 10197）；ISSCAN 用于性能评估 | exact 与 connectivity 层面都从 external 阻断；互变异构体另报敏感性而非混入主结论 | 该设计减少明显结构重叠对 external 估计的影响；外部 AUROC 0.719 低于 validation 0.766，但该差异不能仅归因于隔离策略，也不能与论文数值直接比较 |
| 模型选择 | 论文报告先对 18 个“模型×表征”进行 ISSCAN 评估，并将高表现的 LGBM-RDKit、RF-E-state、BiLSTM-MACCS 用于后续 consensus（论文 §3.1，pp. 10200–10201；§3.2，p. 10202）。其 Table 1 的 ISSCAN LGBM-RDKit accuracy/MCC/AUROC 为 0.800/0.615/0.882，human set（73）为 0.753/0.535/0.842（论文 §3.2，p. 10201） | 候选与选择规则在读取 external 前锁定，以 fixed validation AUROC 选出 LGBM+descriptors；外部只做一次终点评估 | 减少 primary external 驱动的模型选择，结论可审计；代价是不能保证对已知 external 最优。由于任务数据、标签、样本组成、候选模型和评估协议均不同，v1 external MCC 0.288 不能被解释为与论文 MCC 的直接优劣比较 |
| 特征与解释 | 论文使用 ECFP、E-state、MACCS、physchem、RDKit、token 等六表征；以 BiLSTM-MACCS 做 SHAP/子结构和 scaffold 解读 | v1 descriptor 模型保留 220 个 RDKit/physchem descriptor（最终 209），优先做化学性质方向解释；v2 用 ECFP4/MACCS 补足结构视角 | descriptor SHAP 更容易关联尺寸、疏水性、芳香性和极性；代价是少了直接的 bit/子结构解释，且 descriptor 不能直接提供机制因果 |
| 共识模型 | LGBM-RDKit、RF-E-state、BiLSTM-MACCS 的 unanimity 规则，用于 IARC 2B 等应用（论文 §3.5–3.6，pp. 10204–10205） | LGBM-descriptor、RF-MACCS、RF-ECFP4；固定 output contract/policy 后在新的 NTP external 上一次性评估 | external 上 coverage 65.3%、covered MCC 0.647、错误富集明显，但 covered sensitivity 0.625 未达 0.65；因此是 `PARTIAL_SUCCESS`，不能称优于 v1 或可部署 |
| 适用域 | 文献以 kNN 距离寻找 within-domain，并用于 IARC 2B 筛查（论文 §2.10，p. 10199；§3.4，pp. 10203–10204） | 使用最大 ECFP4 Tanimoto、Murcko scaffold、最近邻标签关系和 tautomer 关系；当前只做描述性分层 | 在较大分层中观察到 out-of-domain 的点估计较低（external MCC 0.229）；但未给置信区间，AD 阈值也未在 development 预注册并外部验证，不能直接作为部署拒答规则 |

因此，本项目对原论文最重要的继承不是复刻每一个模型超参数，而是继承“多视角、共识、适用域、解释”的研究框架；最重要的改动是把**数据谱系、冲突裁决、外部隔离和冻结评估**提升为与模型本身同等重要的实验设计。本项目采用更严格且可审计的数据边界，同时在组成不同的 external 集上观察到低于论文报告的指标；由于数据来源、标签、样本组成、候选模型和评估协议均不同，不能把指标差异单独归因于隔离策略，也不能进行直接的模型优劣比较。

## 5. 后续研究与工作流迁移

这套工作流可迁移的核心是：多来源端点统一 → 结构标准化 → 标签冲突治理 → 角色隔离与泄漏审计 → 冻结模型/外部终点评估 → 解释、适用域和不确定性输出。下面区分当前致癌性项目的后续研究与向新端点的迁移，避免把部署延伸混同为 endpoint 迁移。

### 5.1 当前致癌性项目的后续研究

| 方向 | 已有基础 | 必须在任何新 external 前完成的工作 |
|---|---|---|
| 已封存 v2 | final policy、artifact、NTP external、最终 `PARTIAL_SUCCESS` 和 complete audit | 不得再次读取 NTP candidate、重跑、调阈值或基于结果改写 v2；若做新研究必须新建版本和新 external |
| 新 independent external / 后续共识 | v1/v2 的 external lock、manifest、审核与一次性评估链路 | 应作为 v3 或独立研究，重新预注册 label ontology、成员、AD、coverage 和新的未触碰 external；不能把 NTP/CCRIS 作为新的确认集 |
| IARC 2B / NTP RoC 筛查 | v2 的 inconclusive、AD 描述和错误富集证据 | 仅能作为新研究的人工复核/描述性框架；需定义漏检成本、AD、复核触发条件和新的验证集，不能把 frozen v2 直接部署 |
| 风险-覆盖与人工复核政策 | 已有模型概率、分歧、近邻、scaffold 与 descriptor 误差剖面 | 用 development 预注册 selective-risk、漏检成本与人工复核容量；不以 v1 primary external 事后优化这些规则 |

### 5.2 迁移到新 endpoint

| 迁移方向 | 可复用部分 | 新增/需重做部分 | 建议的首个里程碑 |
|---|---|---|---|
| Ames mutagenicity | release/manifest、结构标准化、重复与泄漏审计、ECFP/MACCS/descriptor 特征 | 端点标签规则、菌株/代谢活化等实验条件的角色化 | 先建立按来源和实验条件可追溯的二分类 benchmark |
| Skin sensitization | 冲突审计、外部锁、共识与拒答框架 | OECD assay 的多阶段/多类别标签和物种/试验条件 | 研究二分类与多任务/分层标签哪种更稳定 |
| hERG / cardiotoxicity | 特征、LightGBM/RF、适用域和错误剖面 | 活性阈值、测定条件、连续 IC50 与分类标签的转换策略 | 严格规定 assay 阈值及近阈值不确定样本处理 |
| Hepatotoxicity / DILI | 不确定输出、来源冲突处理、外部评估 | 人体病例、药物暴露、因果等级等异质证据 | 先把“药物相关性”与“剂量/暴露不足”从标签中分离 |
| 急性毒性和生态毒性 | endpoint-agnostic release 框架、scaffold CV、化学空间诊断 | 物种、暴露时长、单位、LC50/LD50 连续值 | 从一个物种/一个统一单位的单任务模型开始 |
| 材料或环境安全筛查 | 可审计结构/数据发布与多来源合并方法 | UVCB、聚合物、盐、纳米材料的结构表示边界 | 先单独定义“可建模结构实体”的纳入/排除策略 |

选择新 endpoint 时，先检查：标签能否形成清晰 ontology；是否存在未参与开发的独立 external；assay 条件异质性是否可管理；有效样本量是否适合本地训练；现有代码能复用多少；以及能否提出不只是“再训练一个分类器”的研究问题。若目标是先验证软件与数据治理骨架的迁移性，Ames 更合适；若目标是练习连续活性、阈值选择与 assay harmonization，hERG 更合适。

### 5.3 当前项目的主要限制

1. fixed validation 是随机分层划分，不是完全 structure-disjoint；其保留的 connectivity-level 跨 split 关系限制了其对新骨架外推的代表性。
2. development 样本只有 921 个；候选排序、分层指标和小样本适用域点估计均具有不确定性。
3. 标签来自异质数据库；来源记录数不等于独立实验或独立证据，自动冲突裁决可能受高记录密度来源影响。
4. v1 primary external 仅来自 CCRIS，虽在 exact InChIKey/connectivity 层面与 development 隔离，却不能代表所有真实应用分布，也未强制消除 tautomer、scaffold 或高相似近邻关系。
5. 当前 external 指标没有置信区间、bootstrap 或正式差异检验，不能对 validation–external 落差作强因果归因。
6. v2 已有独立 NTP external，但只有 49 个样本、covered positive 仅 8 个；sensitivity 的 Wilson 区间很宽。covered 指标不能与全体样本指标直接比较，且 carcinogen coverage（0.533）明显低于 noncarcinogen coverage（0.706）。
7. AD 分箱和阈值目前是描述性分析，未按部署政策预注册或独立验证；最小分箱的完美点估计不稳定。
8. SHAP、descriptor 和错误剖面只反映训练数据中的关联，不是致癌机制或因果解释。
9. 二元标签压缩了剂量、物种、暴露途径、实验条件和人类/动物证据层级等信息。
10. 当前模型仅适用于研究性辅助筛查，不能直接用于临床、监管审批或个体安全决策。

推荐的近期顺序是：

1. 不再触碰已封存 v1 的选择和外部结论；把 v1 总结作为可信基线引用。
2. 不再推进已封存的 v2；若研究共识/拒答策略，应以 v3 或独立新项目重新预注册并使用新的未触碰 external，且将阳性 coverage 与 sensitivity 作为一级风险。
3. 依照上述选择标准选一个相邻端点，重跑数据层而不是复用致癌性标签或 external；用同一 release/审核骨架验证工作流的可迁移性。
